This repository uses a **modular licensing structure** to ensure all tools, data, and biological assets remain permanently in the open-source public domain.

## 1. Software and Source Code

All Python scripts, algorithms, and source code (`.py` files) in this repository are licensed under:

**GNU Affero General Public License v3.0 (AGPL-3.0-or-later)**

This includes but is not limited to:
- All M4 pipeline scripts (`m4_*.py`)
- Validation tools
- Automation controllers
- Analysis utilities

### Why AGPL-3.0?

The AGPL is the most aggressive open-source software license available. Standard licenses (MIT, Apache, GPL) have a "cloud loophole" that allows companies to use the code on private servers without sharing modifications. **The AGPL closes this loophole.** If any entity uses this code over a network (including internal corporate networks), they are legally obligated to open-source all their modifications under the same terms.

**Full license text:** https://www.gnu.org/licenses/agpl-3.0.en.html

## 3. Biological Assets (Prior Art)

All generated peptide sequences, FASTA files, PDB structural models, and molecular designs are released under:

**Open Material Transfer Agreement (OpenMTA)**

This includes:
- All sequences in `.fasta` files
- Structural predictions (`.pdb`)
- Peptide designs for therapeutic applications:
  - D2R-selective agonists (prolactinoma)
  - CFTR chaperones (cystic fibrosis)
  - Antimicrobial peptides (AMP)
  - Cas9 minimization variants
  - THz disaggregation targets

### Why OpenMTA?

Biological sequences cannot be copyrighted (like mathematical equations). However, by attaching the OpenMTA, we:

1. **Clearly designate** these sequences as open biological assets
2. **Reinforce their status** as public domain Prior Art
3. **Destroy future patent claims** by establishing documented publication dates

### Prior Art Statement

All biological sequences published in this repository as of **April 20, 2026** constitute **Prior Art** under:
- 35 U.S.C. § 102 (United States)
- Article 54 EPC (European Patent Convention)
- Section 29 of the Patents Act 1977 (United Kingdom)
- Equivalent provisions in other TRIPS-compliant jurisdictions

Any patent application filed after this date that claims these sequences or obvious derivatives thereof is **invalid** due to lack of novelty.

**OpenMTA text:** https://biobricks.org/openmta/

## Anti-Enclosure Clause

This licensing structure is **intentionally designed** to prevent patent enclosure of open science. By using:

- **AGPL** for the computational factory
- **CC BY-SA** for the blueprints
- **OpenMTA** for the medicine

We ensure that any entity attempting to patent, enclose, or restrict access to these innovations will face:

1. **Legal liability** for license violation
2. **Prior art challenges** to any patent applications
3. **Community enforcement** through the open-source ecosystem
